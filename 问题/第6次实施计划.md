# 底层可信度整改 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复登录测试、权限到人、硬编码与统计口径问题，把项目底层可信度提升到答辩可自证的水平。

**Architecture:** 先固化认证与权限边界，再清理初始化与脚本中的硬编码，最后统一统计口径与测试，避免在业务功能继续扩展前把不稳定底座继续放大。整改优先以后端可验证性为主，前端只做必要同步与说明修正。

**Tech Stack:** Spring Boot 3, Spring Security, MyBatis-Plus, JUnit 5, MockMvc, Vue 3, Vite
