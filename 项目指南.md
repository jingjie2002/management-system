# 项目指南

## 1. 项目名称
基于 SpringBoot 的企业员工精细化人事管理系统

## 2. 技术栈
- 后端：Spring Boot、Spring Security、MyBatis-Plus、H2、MySQL
- 前端：Vue 3、Vite、Element Plus、ECharts
- 构建：Maven Wrapper、npm

## 3. 当前完成情况
- 已实现“入、转、离、考、绩”相关核心能力中的主要业务链路。
- 已实现公告、组织、员工、考勤、绩效、培训、项目、用户管理等基础模块。
- 已补充“转”相关功能，包括岗位调动、职级调整、异动申请、审批、生效、个人中心联动。
- 已完成后端权限补强、注册默认角色分配、统计逻辑修正、关键前端硬编码清理。

## 4. 本地验证结果
- 后端全量测试：`mvnw.cmd test` 通过。
- 前端构建验证：`npm run build` 通过。
- 当前项目目录已完成交付清理，仅保留源码、运行测试文件、文档、`问题` 目录和论文文件。

## 5. 项目目录
- `src`
  后端源码
- `frontend`
  前端源码
- `mvnw`、`mvnw.cmd`
  Maven Wrapper
- `start_all.bat`
  统一一键启动脚本（唯一入口）
- `问题`
  开发过程记录与整改资料
- `基于SpringBoot的企业员工精细化人事管理系统设计与实现-下载评阅稿.docx`
  论文文件

## 6. 运行环境
- Java 17 及以上
- Node.js 18 及以上
- Windows 10/11

## 7. 默认账号
- `admin / 123456`
- `hr / 123456`
- `employee / 123456`

## 8. 建议启动方式
- 快速演示：直接运行 `start_all.bat`
- 答辩展示：优先使用 H2；如需 MySQL，先准备数据库后仍运行 `start_all.bat`

## 9. 重要说明
- 第一次运行前，前端需要先执行 `npm install`
- 如果只想验证代码正确性，先执行：
  - `mvnw.cmd test`
  - `cd frontend && npm run build`
- 如需切换 MySQL，可配置 `DB_URL`、`DB_USERNAME`、`DB_PASSWORD` 环境变量
